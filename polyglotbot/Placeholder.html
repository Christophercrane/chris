Introduction and Overview

Polyglot Bot (Poly) enables skills to be consumed in a wide variety of languages by sending user utterances and responses to the Watson Language Translator service through a Cloud Function. This run book will walk through:

                                               i.     Gathering Requirements

                                              ii.     Assembling the components

                                            iii.     Configuring Watson Assistant

                                            iv.     Patterns and other considerations

 

The intended user experience for Poly enabled Assistants should be seamless. The end user selects a language and is able to interact with the Assistant in the language of their choice. The example below shows a user asking for and receiving an address to an establishment in Spanish despite the fact that the Assistant was configured and trained only in English.

 

 

The user experience is enabled with the following component flow:

 

This flow is made possible by linking Watson Assistant to a custom application that sits in Cloud Functions. The Cloud Function is invoked by a webhook from Watson Assistant. Once invoked, the Cloud Function leverages both Language Translator and Assistant to correctly respond to the user.

2.    Create Cloud Function: Cloud Functions is a serverless programming platform for developing code that scalably executes on demand. We will be leveraging this tool because it integrates with Watson Assistant via a Webhook and carries low costs.

                                               i.     Create an instance of Cloud Functions if you haven’t already

                                              ii.     Create a Cloud Function and note the name

                                            iii.     Login to your IBM Cloud from the CLI using the following command:

ibmcloud login –sso

                                            iv.     Choose the target space:

ibmcloud target --cf

                                              v.     Place your polyglot.zip file in its own folder on your computer where you can find it.

                                            vi.     Using IBM Cloud CLI Locate the folder containing your zip file (polyglot.zip).

                                           vii.     From within the folder, create your cloud function with the following command from the CLI:

ibmcloud fn action create [your cloud function name here] polyglot.zip --kind nodejs:10

                                         viii.     Once the creation is complete you should get a confirmation node. You can always update this file later if necessary with the following command:

ibmcloud fn action update [your cloud function name here] polyglot.zip --kind nodejs:10

                                            ix.     Once you have logged into the IBM Cloud CLI navigate to the folder holding the downloaded zip file and enter the following:

ibmcloud fn action create my-action my-action.zip --kind nodejs:10

                                              x.     This command will push the code to the cloud function and make it available for your skill.
